# Proyecto Final

## Propósito del proyecto
El proyecto final integra las competencias del curso (limpieza de datos, análisis exploratorio de datos – EDA, y visualización) mediante el estudio de un problema contemporáneo asociado al bienestar digital y la experiencia de las nuevas generaciones en entornos altamente conectados (p. ej., uso de redes sociales, tiempo de pantalla, sueño, rendimiento académico, ansiedad/estrés autopercibido, hábitos de actividad física y patrones de comunicación). El enfoque del proyecto se inspira en las preguntas y tensiones planteadas en *The Anxious Generation*, utilizando evidencia basada en datos para construir un diagnóstico y proponer intervenciones.

## Modalidad de trabajo
* El proyecto se realizará en equipos de 2 o 3 estudiantes.
* La complejidad esperada del análisis y de los datos aumenta con el tamaño del equipo: a mayor número de integrantes, mayor alcance técnico, mayor variedad de fuentes y mayor profundidad analítica.

## Producto final evaluable
Cada equipo entregará:
1. un análisis reproducible (notebook y/o repositorio con código, datos y README),
2. un informe técnico (breve y estructurado), y
3. una presentación oral con visualizaciones, interpretación de resultados, recomendaciones y la solución propuesta.

## Complejidad mínima según tamaño del equipo

### Equipos de 2 estudiantes
* Integrar **al menos 2 fuentes de datos** (datasets distintos).
* Realizar un pipeline completo: limpieza → EDA → visualizaciones → conclusiones y propuesta.

### Equipos de 3 estudiantes
* Integrar **al menos 3 fuentes de datos** (datasets distintos) y justificar la necesidad de cada una.
* Incluir **una etapa adicional de complejidad**, por ejemplo:
    * unión de datos con distintas granularidades (diario/mensual; individual/regional),
    * construcción de variables derivadas (índices, tasas, normalizaciones),
    * segmentación comparativa (por edad, género, región, cohorte, tipo de uso),
    * análisis de sensibilidad o validaciones (checks, escenarios, comparación de definiciones).

## Pregunta de investigación
Cada equipo debe formular una pregunta investigable inspirada en el tema del libro (bienestar digital, hábitos de pantalla, redes sociales, sueño, rendimiento, salud mental, aislamiento social, etc.).

La pregunta debe cumplir con estas condiciones:
* Ser concreta y medible (debe poder responderse con variables observables).
* Tener una unidad de análisis definida (personas, estudiantes, escuelas, regiones, países, periodos).
* Requerir la integración de múltiples fuentes de datos para construir evidencia (no solo “un dataset y ya”).
* Permitir llegar a una propuesta de solución de ingeniería basada en hallazgos.

## Solución del problema evaluado
El proyecto no debe limitarse a describir patrones en los datos. Cada equipo deberá identificar un problema concreto, medible y relevante, construir evidencia mediante el análisis de datos y proponer una solución o recomendaciones viables basada en los hallazgos obtenidos.

## Calidad de los datos
Es importante que cada equipo se asegure del proceso de calidad de datos antes de presentar sus hallazgos. Este proceso debe demostrar que los datos utilizados son adecuados, consistentes y confiables para responder la pregunta de investigación planteada.

Por ejemplo, incluir:
* Revisión de valores faltantes, duplicados, errores de formato y valores atípicos.
* Validación de tipos de variables, unidades de medida, rangos esperados y consistencia entre fuentes.
* Justificación de las decisiones de limpieza, transformación, imputación o eliminación de registros.
* Verificación de la coherencia entre las distintas fuentes integradas.
* Registro claro de los cambios realizados sobre los datos originales para asegurar trazabilidad y reproducibilidad.
* Evaluación de las limitaciones de los datos y de cómo estas afectan la interpretación de los resultados.

Para este proyecto se evaluaran los dos resultados de aprendizaje de la asignatura:

### Resultado de aprendizaje 1: Diseño de soluciones de Ingeniería

**Criterio de evaluación**
El estudiante demuestra capacidad para diseñar una solución de ingeniería basada en evidencia a partir de hallazgos del análisis, definiendo un problema medible, restricciones y criterios de éxito, y proponiendo una intervención viable.

### Resultado de aprendizaje 2: Ingeniería práctica nivel avanzado

**Criterio de evaluación**
El estudiante demuestra dominio de ingeniería práctica avanzada al construir un flujo de trabajo de análisis robusto, reproducible y mantenible, aplicando buenas prácticas de tratamiento de datos, control de calidad, trazabilidad y comunicación visual efectiva.
